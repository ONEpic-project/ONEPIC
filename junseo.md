3. AI 딥러닝 구조 설계
3.1 왜 단일 YOLO 모델만 사용하지 않았는가?

YOLO는 객체의 위치를 빠르고 정확하게 탐지하는 데 매우 강력한 모델이지만,
다음과 같은 한계가 존재한다.

용량만 다른 상품 (예: 350ml vs 500ml)

맛/패키지가 유사한 상품

미세한 텍스트/색상 차이가 중요한 경우

이러한 Fine-grained Classification 문제에서는
단일 Detection 모델만으로 높은 정확도를 보장하기 어렵다.

3.2 OCR을 사용하지 않은 이유

OCR은 다음과 같은 용도에 적합하다.

가격표 인식

유통기한, 진열대 텍스트 인식

그러나 상품 패키지는:

곡면

반사

왜곡

브랜드 디자인 위주의 텍스트

특성으로 인해 상품 정체성 인식에는 OCR이 부적합하다.

따라서 본 프로젝트에서는 OCR 대신
딥러닝 기반 이미지 특징 학습 방식을 채택하였다.

4. 2-Stage AI 인식 구조
4.1 1단계: YOLOv8 객체 탐지

모델: YOLOv8 Detect

역할:

이미지 내 상품 위치 탐지

1차 클래스 후보 예측

출력:

Bounding Box 좌표

클래스 ID

Confidence Score

YOLO는 “어디에 무엇이 있는가”를 빠르게 판단하는 역할을 수행한다.

4.2 2단계: 추가 딥러닝 분류 모델

모델 계열: MobileNet / EfficientNet-Lite 계열

최종 사용 모델: EfficientNet-Lite 기반 경량 분류 모델

역할:

YOLO가 탐지한 상품 영역(Crop)을 입력으로 받음

유사 상품 간 미세 차이 학습

최종 상품 정체성 결정

이 모델은 모바일 및 CPU 환경에서도 추론이 가능하도록 설계되었다.

4.3 왜 이 구조가 실무적으로 타당한가?

이 구조는 실제 리테일 및 스마트 비전 시스템에서 널리 사용되는 방식이다.

객체 탐지: 위치/대상 분리

분류 모델: 정체성 정확도 강화

역할 분리로 모델 간 책임 명확

이를 통해 YOLO 단독 대비 정확도를 유의미하게 향상시킬 수 있다.
